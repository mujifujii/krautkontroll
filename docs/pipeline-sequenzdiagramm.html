Krautkontroll – Continuous-Delivery-Pipeline (Sequenzdiagramm)

Pipeline-Sequenzdiagramm für Teil 1. Bezieht Entwicklung, Qualitätssicherung (QS) und die verwendeten Protokolle (an den Pfeilen in [eckigen Klammern]) ein. Diese Datei einfach im Browser öffnen – das Diagramm wird automatisch gerendert. 

sequenceDiagram
    actor Dev as Entwickler
    participant Local as Lokale Umgebung (Git, Maven)
    participant GitHub as GitHub (Remote-Repo)
    participant CI as GitHub Actions (CI/CD)
    participant Prod as GHCR (Produktiv-Registry)

    Note over Dev,Local: Entwicklung (lokal)
    Dev->>Local: Code ändern
    Dev->>Local: ./mvnw verify (lokal testen)
    Local-->>Dev: Testergebnis (grün/rot)
    Dev->>Local: git commit

    Note over Dev,CI: Continuous Integration (Must)
    Dev->>GitHub: git push [HTTPS/SSH]
    GitHub->>CI: Workflow auslösen – Webhook [HTTPS]
    CI->>GitHub: Quellcode auschecken [HTTPS]

    Note over CI: Qualitätssicherung:
kompilieren · Unit-Tests (surefire) ·
Integrationstests (failsafe) · JaCoCo-Coverage ·
Artefakte ablegen

    alt Pipeline erfolgreich
        CI-->>GitHub: Status "success" [HTTPS]
        GitHub-->>Dev: Benachrichtigung "grün" [HTTPS/SMTP]
    else Pipeline fehlgeschlagen
        CI-->>GitHub: Status "failure" [HTTPS]
        GitHub-->>Dev: Benachrichtigung "rot" [HTTPS/SMTP]
    end

    Note over CI,Prod: Continuous Deployment (Could) – nur auf main
    opt grüne Pipeline und Branch = main
        CI->>CI: Docker-Image bauen
        CI->>Prod: Image pushen [HTTPS]
        Prod-->>CI: Image veröffentlicht
    end

    Note over Dev,CI: Monitoring (Could) – zeitgesteuert (cron, alle 30 Min)
    loop alle 30 Minuten bzw. manuell
        CI->>CI: Anwendung starten
        CI->>CI: GET /actuator/health [HTTP]
        alt Status = UP
            CI-->>Dev: OK (grüner Lauf)
        else nicht erreichbar
            CI-->>Dev: Alarm "rot" [HTTPS/SMTP]
        end
    end

Legende der Protokolle

ProtokollWo verwendet
HTTPS / SSHgit push von lokal zu GitHub
HTTPSWebhook-Trigger, Checkout, Status-Updates, Image-Push zu GHCR
HTTPHealthcheck-Abfrage /actuator/health im Monitoring
SMTPE-Mail-Benachrichtigung über das Lauf-Ergebnis (über GitHub)
